# План автоматизации тестирования

### Позитивные сценарии:

**Оплата "Валидной" картой:**

- Воспроизведение:

1. Открыть страницу "покупки тура"
2. Нажать кнопку "Купить"
3. Заполнение в появившихся полях валидный данных пользователя
4. Нажать кнопку "Продолжить" 

- Результат:

1. Данные успешно отправлены и отображаются в базе данных.
2. Появляется окно об одобрении проведенной операции банком.

**Оплата "Валидной" картой в кредит:**

- Воспроизведение:

1. Открыть страницу "покупки тура"
2. Нажать кнопку "Купить в кредит"
3. Заполнение в появившихся полях валидный данных пользователя
4. Нажать кнопку "Продолжить"

- Результат:

6. Данные успешно отправлены и отображаются в базе данных.
7. Появляется окно об одобрении проведенной операции банком.

### Негативные сценарии:
(Аналогичные сценарии для покупки тура и покупки тура в кредит)

**Оплата с пустой формой:**

- Воспроизведение:

1. Открыть страницу "покупки тура"
2. Нажать кнопку "Купить"
3. Появившиеся поля оставить пустыми
4. Нажать кнопку "Продолжить"

- Результат:

6. Данные не отправлены.
7. Под полями появляется сообщение об ошибке: 
- "Номер карты - "Неверный формат" 
- "Месяц" - "Неверный формат"
- "Год" -"Неверный формат"
- "Владелец" - "Поле обязательно для заполнения"
- "CVC/CVV" - "Неверный формат"

**Оплата с неполными данными в поле "Номер карты":**

- Воспроизведение:

1. Открыть страницу "покупки тура"
2. Нажать кнопку "Купить"
3. В поле “Номер карты” ввести неполный номер карты (ввести номер карты без 1го символа) . Остальные поля заполнить валидными данными.
4. Нажать кнопку “Продолжить”.

- Результат:

1. Данные успешно отправлены и отображаются в базе данных.
2. Появляется окно об отказе проведенной операции банком.
